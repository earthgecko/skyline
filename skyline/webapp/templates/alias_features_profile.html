Alias :: features profiles

You can alias existing features profiles to other metrics. This allows you to assign features profiles for a certain namespace to a new namespace. For example say you deploy a new bigger server to move some service/s or role/s too, the server metrics themselves may be different but the application or service metrics may be expected to be the same or similar when the service/role is migrated onto the new server. Before aliasing features profiles from one namespace to another the metrics to be aliased must exist first.

Option value
metric_pattern The metric pattern The metric pattern for the features profiles you want to alias to another namespace, this is the src pattern e.g. skyline-server-1 It is possible to use a single * wildcard in the metric_pattern e.g. skyline.*.skyline-server-1, the pattern must be dot separated but the dots are not regex dots, they are namespace element separators. With labelled metrics you must still use a dotted metric pattern so for instance to match: vm_rows_added_to_storage_total{alias="VictoriaMetrics", cluster="victoriametrics-cluster-1-eu", component="vmstorage", instance="victoriametrics-prod:8482", job="victoriametrics-cluster-1-eu-vmstorage"} You could use a metric_pattern of vm_rows_added_to_storage_total.victoriametrics-prod-1:8482 with labelled metrics, each metric_name, label and value is considered a namespace element.
replace_pattern The replace pattern The replacement/alias pattern to alias features profiles to, this is the destination pattern e.g. skyline-server-2 The use of a wildcard and labelled metrics are handled the same in replace_pattern as specified above for metric_pattern.
